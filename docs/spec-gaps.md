# Spec Gaps — Rules and Constraints Not in the Original Specification

This file documents constraints that emerged during Phase 2 build that were not captured in the original Phase 2 spec. Each entry explains what was missing, what rule was established to fill the gap, and where it now lives.

---

## Image SEO Rules — Not in original spec

**Gap discovered:** 2026-05-15
**Gap:** The original Phase 2 spec (`Phase2_Kickoff_Homepage_ServicePages.md`) and `BUILD_PLAN.md` contained no guidance on image filenames, alt text requirements, OG image generation, favicon treatment, or the prohibition against `<img>` tags. Images were initially saved with generic names (e.g., `hero-home.jpg`, `image1.jpg`) before the gap was identified.

**Rule established:** Six image SEO rules added to CLAUDE.md under `## IMAGE SEO RULES — ENFORCE THROUGHOUT`:
1. Next.js `<Image>` required everywhere — no `<img>`
2. Keyword-aware filenames in `public/images/` using hyphens
3. Alt text required on every non-decorative image; must describe the photo accurately
4. OG image generated by `src/app/opengraph-image.tsx` — no hardcoded OG URLs
5. Favicon generated by `src/app/icon.tsx` — no manual `<link rel="icon">` tags
6. No external image hotlinks — all images must be local

**Impact:** Eight service card images were renamed on 2026-05-15. Hero image renamed from `hero-home.jpg` to `chicago-office-furniture-installation.jpg`.

## Voice and Content Rules — Insufficient detail in original spec

**Gap discovered:** 2026-05-15
**Gap:** The original spec included a brief "Voice and tone" paragraph ("Direct, plain-spoken, professional but approachable. Speaks to B2B buyers.") but did not enumerate specific prohibited words, phrases, or punctuation. This allowed generated copy to include em dashes, corporate buzzwords ("leverage", "optimize", "white-glove"), and unverified claims to slip through the initial build.

**Rule established:** Detailed `VOICE AND CONTENT RULES` section added to CLAUDE.md with specific prohibitions:
- No em dashes
- No title-case compound adjectives ("On-Time Delivery", "Client-Focused Teams")
- No corporate plurals ("solutions", "partnerships", "offerings", "deliverables")
- No consultant verbs ("leverage", "optimize", "synergize", "enable", "facilitate")
- No "white-glove"
- Contractions required where natural
- Voice test: would a 15-year contractor actually say this?

**Also established:** Sourcing rule requiring all copy to trace to one of three verified sources (audit spec, live site, Brian documents). Anything else gets a placeholder and a known-issues.md flag.

## Live-Site Parity Rule — Spec assumed sections without verification

**Gap discovered:** 2026-05-15
**Gap:** The original Phase 2 spec listed page sections for the homepage (Hero, Trust Bar, Services Grid, Why On Point, Service Area, Review Snippet, CTA Block) without verifying whether each section existed on the live site. Several sections were built with invented content before the gap was identified:
- The services grid initially showed 6 invented Chicago-SEO-focused service names that did not match the 8 services on the live site
- Service card descriptions were invented rather than pulled from live service pages
- The "Why On Point" and review snippet sections had not been verified against live-site structure

**Rule established:** `LIVE-SITE PARITY RULE` added to CLAUDE.md:
- Every section in the new build must exist on the live site OR be logged in `docs/spec-additions.md` with source authorization
- Fetch the live page before building any new page
- If a section is in the spec but not on the live site, check `docs/spec-additions.md` for authorization before building it

**Impact:** PRIMARY_SERVICES updated from 6 invented slugs to the exact 8 live-site services. Service card descriptions replaced with verbatim/adapted text from live service pages. The `docs/spec-additions.md` file established as the authorization record for intentional departures.

## Service Architecture — Audit URLs vs. Live-Site Services (RESOLVED 2026-05-16)

**Gap discovered:** 2026-05-16
**Gap:** The original Phase 2 spec specified 6 Chicago-SEO-focused service page slugs (commercial-furniture-installation, cubicle-installation, office-relocation, systems-furniture-installation, office-furniture-delivery-setup, commercial-space-planning). The live site has 8 different services. These two sets did not align, and the prior session's service architecture was wrong.

**Resolution:** New architecture decision locked 2026-05-16:

1. **8 service pages** — matching the live site's 8 services exactly (Office Installations, Relocation, Warehousing, Space Planning, Electrical & Voice/Data, Artwork Installation, Window Treatment Installations, Cubicle Wall and Upholstery Cleaning)
2. **Slug strategy** — audit URLs used where they map cleanly to one of the 8; new Chicago-suffixed slugs used for the remaining 4 (artwork-installation-chicago-il, electrical-voice-data-cabling-chicago-il, window-treatment-installation-chicago-il, cubicle-wall-upholstery-cleaning-chicago-il)
3. **Three audit-only URLs not built as standalone pages** — cubicle-installation-chicago-il, systems-furniture-installation-chicago-il, office-furniture-delivery-setup-chicago-il redirect permanently to the Office Installations money page. Their audit-specified keywords are preserved as H2 sections and FAQ entries within that page.

**What changed in code:**
- `constants.ts` — PRIMARY_SERVICES updated with 4 new Chicago-suffixed slugs
- 4 route directories renamed to new slugs
- 4 retired route directories deleted
- `next.config.ts` — redirects added for all old slugs
- Homepage SERVICE_DESCRIPTIONS keys updated
- See `docs/content-source-map.md` for the canonical source reference for all 8 pages
